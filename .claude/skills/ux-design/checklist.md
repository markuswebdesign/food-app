# UX Design Checklist

Before handing off:

## Goals
- [ ] Primary user job stated in one sentence
- [ ] Success metric stated
- [ ] Non-goals listed

## Flows
- [ ] Happy path step-by-step
- [ ] Entry points + completion state
- [ ] Edge paths covered (empty, validation, network, auth, partial)

## IA / Screens
- [ ] Each screen has clear responsibility and actions
- [ ] Information hierarchy supports decision-making

## Interactions
- [ ] Loading/success/error feedback specified
- [ ] Confirmation/undo rules specified
- [ ] Keyboard expectations noted where relevant

## Copy
- [ ] Actionable empty states
- [ ] Specific error messages
- [ ] Consistent terminology with product
